# Ark Station 2.1 (Lab 4)

Вторая итерация симулятора управления космической станцией. Основной фокус работы — перегрузка операторов, декомпозиция кода и безопасное управление динамической памятью.

## Новые механики (Lab 4)

### 1. Перегрузка операторов
- **I/O (`<<`)**: Реализован дружественный оператор вывода для `Robot` и `Module`. Теперь информация выводится в виде форматированных карточек.
- **Жизненный цикл (`++`, `--`)**: 
  - `--robot` (префиксный): Ежедневный износ (шасси/прошивка -10%).
  - `++robot` (префиксный): Принудительный ремонт (за 50 Э / 50 Д).
- **Синтез сознания (`+`)**: Создание нового робота путем скрещивания двух родителей (старше 5 дней). Характеристики рассчитываются с учетом мутации (0.8 - 1.2).
- **Модульная сборка (`+`)**: Слияние двух одинаковых модулей одного уровня для получения модуля следующего уровня.

### 2. Вариант 2: Детектор Клонов (`==`)
Реализован оператор сравнения роботов. Система проверяет идентичность:
- Типа (фракции);
- Суммарной производительности (эффективности);
- Текущего состояния (HP).
Используется для поиска ошибок дефрагментации на станции.

### 3. Интеграция Варианта 8 (Lab 3)
Сохранен уникальный робот «Локальный вычислитель», который производит ресурсы, но не нагружает общую сеть связи станции (Power Contribution = 0).

## Архитектура
Проект полностью отрефакторен:
- **Декомпозиция**: Принцип «1 класс — 1 файл». Всего 11 пар `.h` / `.cpp`.
- **Инкапсуляция**: Все поля данных `private`.
- **Полиморфизм**: Использование виртуальных деструкторов и `override` методов.
